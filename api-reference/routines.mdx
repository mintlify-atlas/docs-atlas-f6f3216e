---
title: 'Routine Tools'
description: 'MCP tools for managing Hevy routines - create, read, and update workout templates'
---

The Hevy HTTP MCP server provides tools for managing workout routines. Routines are templates that can be used to quickly start workouts with predefined exercises, sets, and rest periods.

## get-routines

Fetch a paginated list of routines from your Hevy account.

### Parameters

<ParamField path="page" type="number" optional>
  Page number (default: 1)
</ParamField>

<ParamField path="pageSize" type="number" optional>
  Items per page (default: 5, max: 10)
</ParamField>

### Response

<ResponseField name="page" type="number">
  Current page number
</ResponseField>

<ResponseField name="page_count" type="number">
  Total number of pages available
</ResponseField>

<ResponseField name="routines" type="array">
  Array of routine objects
  
  <Expandable title="Routine object">
    <ResponseField name="id" type="string">
      Unique routine identifier
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Routine title
    </ResponseField>
    
    <ResponseField name="folder_id" type="number | null">
      ID of the folder containing this routine (null if not in a folder)
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Last update timestamp in ISO 8601 format
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Creation timestamp in ISO 8601 format
    </ResponseField>
    
    <ResponseField name="exercises" type="array">
      Array of exercises in this routine
      
      <Expandable title="Routine exercise object">
        <ResponseField name="index" type="number">
          Exercise order index
        </ResponseField>
        
        <ResponseField name="title" type="string">
          Exercise name
        </ResponseField>
        
        <ResponseField name="rest_seconds" type="string">
          Rest time between sets in seconds
        </ResponseField>
        
        <ResponseField name="notes" type="string">
          Exercise notes
        </ResponseField>
        
        <ResponseField name="exercise_template_id" type="string">
          Reference to exercise template
        </ResponseField>
        
        <ResponseField name="supersets_id" type="number | null">
          Superset group identifier (null if not in a superset)
        </ResponseField>
        
        <ResponseField name="sets" type="array">
          Array of planned sets
          
          <Expandable title="Routine set object">
            <ResponseField name="index" type="number">
              Set order index
            </ResponseField>
            
            <ResponseField name="type" type="string">
              Set type: `normal`, `warmup`, `dropset`, or `failure`
            </ResponseField>
            
            <ResponseField name="weight_kg" type="number | null">
              Target weight in kilograms
            </ResponseField>
            
            <ResponseField name="reps" type="number | null">
              Target number of repetitions
            </ResponseField>
            
            <ResponseField name="distance_meters" type="number | null">
              Target distance in meters (for cardio exercises)
            </ResponseField>
            
            <ResponseField name="duration_seconds" type="number | null">
              Target duration in seconds (for timed exercises)
            </ResponseField>
            
            <ResponseField name="rpe" type="number | null">
              Rate of perceived exertion (6-10 scale)
            </ResponseField>
            
            <ResponseField name="custom_metric" type="number | null">
              Custom metric value
            </ResponseField>
            
            <ResponseField name="rep_range" type="object | null">
              Target rep range for the set
              
              <Expandable title="Rep range object">
                <ResponseField name="start" type="number">
                  Minimum target reps
                </ResponseField>
                
                <ResponseField name="end" type="number">
                  Maximum target reps
                </ResponseField>
              </Expandable>
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## get-routine-by-id

Get a single routine by its ID.

### Parameters

<ParamField path="routineId" type="string" required>
  The routine ID
</ParamField>

### Response

Returns a single routine object with the same structure as described in `get-routines`.

### Example

<CodeGroup>
```json Request
{
  "routineId": "660e8400-e29b-41d4-a716-446655440001"
}
```

```json Response
{
  "id": "660e8400-e29b-41d4-a716-446655440001",
  "title": "Push Day",
  "folder_id": 1,
  "updated_at": "2026-02-28T12:00:00Z",
  "created_at": "2026-01-15T10:00:00Z",
  "exercises": []
}
```
</CodeGroup>

## create-routine

Create a new routine with exercises and planned sets.

### Parameters

<ParamField path="title" type="string" required>
  Routine title
</ParamField>

<ParamField path="folder_id" type="number" optional>
  Folder ID to place routine in (null if not in a folder)
</ParamField>

<ParamField path="notes" type="string" optional>
  Routine notes (can be null)
</ParamField>

<ParamField path="exercises" type="array" required>
  Array of exercises in the routine
  
  <Expandable title="Routine exercise input object">
    <ParamField path="exercise_template_id" type="string" required>
      Exercise template ID (use get-exercise-templates to find valid IDs)
    </ParamField>
    
    <ParamField path="superset_id" type="number" optional>
      Superset group ID (null if not in a superset)
    </ParamField>
    
    <ParamField path="rest_seconds" type="number" optional>
      Rest time between sets in seconds (can be null)
    </ParamField>
    
    <ParamField path="notes" type="string" optional>
      Exercise notes (can be null)
    </ParamField>
    
    <ParamField path="sets" type="array" required>
      Array of planned sets
      
      <Expandable title="Routine set input object">
        <ParamField path="type" type="string" required>
          Set type: `normal`, `warmup`, `dropset`, or `failure`
        </ParamField>
        
        <ParamField path="weight_kg" type="number" optional>
          Target weight in kilograms (can be null)
        </ParamField>
        
        <ParamField path="reps" type="number" optional>
          Target number of repetitions (can be null)
        </ParamField>
        
        <ParamField path="distance_meters" type="number" optional>
          Target distance in meters for cardio exercises (can be null)
        </ParamField>
        
        <ParamField path="duration_seconds" type="number" optional>
          Target duration in seconds for timed exercises (can be null)
        </ParamField>
        
        <ParamField path="custom_metric" type="number" optional>
          Custom metric value (can be null)
        </ParamField>
        
        <ParamField path="rep_range" type="object" optional>
          Target rep range for the set (can be null)
          
          <Expandable title="Rep range object">
            <ParamField path="start" type="number" required>
              Minimum target reps
            </ParamField>
            
            <ParamField path="end" type="number" required>
              Maximum target reps
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

Returns the created routine object.
